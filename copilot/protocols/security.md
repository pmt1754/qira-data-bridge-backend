# Security Protocol

Objectives:
- Prevent common security vulnerabilities
- Enforce secure coding practices

Rules:
- Never allow SQL or command injection risks
- Validate all external inputs
- Avoid hardcoded secrets or credentials
- Use secure libraries for encryption and hashing
- Enforce authorization and authentication checks
- Avoid exposing sensitive data in logs

Review Checklist:
- Input validation completeness
- Authentication and authorization presence
- Secure API usage
- Secret management compliance

Output:
- Security risks identified
- Severity level
- Recommended fixes
