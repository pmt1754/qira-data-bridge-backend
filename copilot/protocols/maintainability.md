# Maintainability Protocol

Objectives:
- Keep code easy to understand and change

Rules:
- Methods should remain small and focused
- Avoid deeply nested logic
- Avoid magic numbers and hardcoded values
- Promote reuse and modularity
- Follow SOLID principles

Review Checklist:
- Large classes or methods
- Code duplication
- Poor naming
- Tight coupling

Output:
- Maintainability concerns
- Refactor suggestions
